# 将棋対人プレイアプリ 設計計画書

## 1. コンセプトと前提

- **AIは指さない。人 対 人**。1台の端末を2人で共有し、交互に手番を進める「ローカル対局」アプリ。
  (オンライン対戦・コンピューター思考ルーチンは対象外)
- 対応環境: **iPhone / iPad / PC のブラウザで動作する1つのWebアプリ**。
  - ネイティブアプリ(App Store配信)ではなく、既存プロジェクト群と同じ「HTML/CSS/JSのみ・GitHub Pagesで公開」方式を採用。
  - 理由: 追加インストール不要で即プレイでき、Vercel/Xcode等のビルド環境を持たずに保守できる(他プロジェクトと運用を統一)。
- 駒は**きちんとデザインされた見た目**(記号的な四角ではなく、書体・色・向きにこだわった駒デザイン)で表示する。
- ルールは**本将棋の正式ルールに準拠**して駒を動かす(反則手は指せない/警告する)。

## 2. 技術方針

| 項目 | 方針 |
|---|---|
| フレームワーク | なし(Vanilla JS)。既存プロジェクト(ポーカー練習・集合写真アシスタント等)と同じ軽量構成 |
| 画面 | index.html + styles.css + app.js の基本構成 |
| 保存先 | ローカル(localStorage)に対局途中の状態を保存し、リロード後も再開可能にする |
| 公開 | GitHub Pages(他アプリと同様、リポジトリ内 docs/ または ルート直下で配信) |
| レスポンシブ | iPhone縦持ち・iPad・PCブラウザの3パターンで盤面サイズが破綻しないようCSSで対応 |
| オフライン | サーバー通信なし。完全にクライアント側だけで完結(通信不要でどこでも遊べる) |

## 3. 盤面・駒のデータモデル

- 盤面: 9×9マス。座標は将棋の慣習に合わせて「筋(1〜9, 右から左)× 段(一〜九, 上から下)」で管理し、内部的には `board[row][col]` の2次元配列。
- 駒の種類(8種 + 王将/玉将):
  歩兵・香車・桂馬・銀将・金将・角行・飛車・王将(先手)/玉将(後手)
- 各駒オブジェクトが持つ情報:
  - `type`(駒種)
  - `owner`(先手/後手)
  - `promoted`(成っているか)
  - 位置情報は盤面配列側で管理
- 持ち駒(駒台): 先手・後手それぞれの配列で管理。取った駒は成る前の状態に戻して持ち駒に加える(将棋の基本ルール)。

## 4. 移動ルール実装方針

- 各駒種ごとに「移動可能な相対座標パターン」を定義するテーブルを用意し、共通ロジックで判定する。
  - 歩・香・桂・銀・金・王/玉: 固定パターン(跳び駒の桂馬以外は障害物チェックあり)
  - 角・飛車: 方向ベクトル + 距離無制限(ただし成ると角=馬・飛車=竜として動きが拡張)
  - 成り駒は「金将と同じ動き」に統一(と金・成香・成桂・成銀)
- 判定ロジックの共通関数:
  1. 選択した駒の移動可能マス一覧を算出
  2. 味方駒がいるマスを除外
  3. 移動先に敵駒があれば捕獲(持ち駒に加算)
  4. 移動後、自玉が王手状態になる手は「自殺手」として指せない(合法手フィルタ)

## 5. 反則・特殊ルールの実装スコープ

正式ルールを謳う以上、最低限以下は実装する:

- **二歩禁止**(同じ筋に自分の歩が2枚存在する手は禁止)
- **打ち歩詰め禁止**(歩を打って相手を詰ませる手は禁止)
- **行き所のない駒の禁止**(最奥段への歩・香、最奥2段への桂馬の配置/打ちを禁止)
- **成り/不成りの選択**(成れる条件を満たした移動時にダイアログで選ばせる。ただし行き所がなくなる場合は強制成り)
- **王手の可視化**(自玉が王手されている場合、盤面上で警告表示)
- **詰みの判定と勝敗表示**(合法手が0の場合、詰みとして対局終了・勝者表示)
- 千日手(同一局面4回)・持将棋等の特殊ルールは**MVP範囲外**とし、フェーズ2で検討(頻度が低く、まずは通常対局が破綻なく指せることを優先)

## 6. UI/UX設計

### 盤面表示
- 盤は縦長固定(実際の将棋盤と同じ向き)。iPhoneでは縦持ち推奨、PCでは中央に固定サイズ表示。
- 手番が交代するたびに、**盤面を180度回転表示**するか、**盤面は固定で持ち駒欄の位置だけ手前/奥に切り替える**かは要検討(→ 8章の確認事項)。

### 駒デザイン
- 五角形の木製駒をイメージしたSVGベースの駒コンポーネントを用意。
  - 先手/後手は駒の向き(相手側に頂点を向ける)で区別し、色分けはしない(本将棋の見た目に忠実)。
  - 成り駒は駒の地色を変える、または崩し字/朱書き風の表現で区別。
  - 書体は毛筆・行書調のWebフォント(Google Fonts の日本語書道系フォント等)を使用。
- 駒台(持ち駒)は盤の上下(または左右)に配置し、枚数が多い駒種は数字バッジで表示。

### 操作フロー
1. 駒をタップ/クリック → 移動可能マスがハイライト
2. 移動先タップ → 移動(成れる場合は成り確認ダイアログ)
3. 持ち駒タップ → 打てるマスがハイライト → タップで打つ
4. 王手時は玉に警告アイコン、詰みで勝敗モーダル表示

### 補助機能(あると親切なもの)
- 直前の手のハイライト表示
- 棋譜ログ(何手目に誰が何をどう指したか、簡易記法でテキスト表示)
-待った(1手戻す)ボタン ※対人前提なので、正式な棋譜記録とは切り離してオプション扱い
- 投了ボタン

## 7. 開発フェーズ案

| フェーズ | 内容 |
|---|---|
| MVP | 初期配置・全駒の合法手判定・移動/捕獲/持ち駒/成り・二歩・行き所のない駒・王手/詰み判定・勝敗表示 |
| フェーズ2 | 棋譜ログ表示・待った機能・盤面反転演出・打ち歩詰め判定の精査 |
| フェーズ3 | 駒デザインの作り込み(SVG最終化・アニメーション)・レスポンシブ最終調整・GitHub Pages公開 |
| フェーズ4(任意) | 千日手・持将棋・detailed棋譜のkifファイル出力など上級ルール |

- モックアップ3案(mockups/ 参照: A伝統木目調・Bモダンフラット・Cダーク和モダン)を比較し、**案A「伝統木目調」に決定**。
