# Tsume Shogi — プロジェクトメモ

更新日: 2026-07-17（問題を64問に拡充、盤上移動・持ち駒なし5手詰めを重点追加）

## プロダクト方針

- iPhone縦持ちを第一対象にした、ブラウザで遊べる詰将棋学習アプリ。
- PWAとしてホーム画面へ追加でき、初回読み込み後はオフラインでも起動できる。
- 対応言語は日本語・英語・フランス語・スペイン語（2026-07-17に仏西を追加）。翻訳辞書を追加すれば言語を増やせる構造。
  - UI文言: `app.mjs` の `I18N`・`PIECES`。問題文: `puzzle-i18n.mjs`（基本問題のfr/es）。反復問題の文章は自動生成。
  - 全問題が4言語を持つことは `tests/game-core.test.mjs` の言語完全性テストで保証。
- 駒表示は「漢字」「ローマ字」「漢字＋ローマ字」の3方式。
- 外観は「伝統」「モダン」「夜」の3テーマ。
- 進捗、表示言語、駒表示、テーマ、判定方法は端末内にのみ保存する。

## 現在の学習体験

1. 1手詰め・3手詰め・5手詰め・7手詰め・9手詰めの局面を表示する。
2. 盤上の攻め方の駒、または持ち駒を選ぶ。
3. 合法な移動先だけを表示する。
4. 設定の「判定方法」で、次の2方式を選べる（2026-07-17追加）。
   - **1手ごと**: 正解の詰み筋から外れる王手をその場で知らせ、盤面を進めず同じ局面から試し直せる。
   - **通しで挑戦**（初期値）: 王手である着手は受け入れ、手数を使い切って詰まなかったとき、または王手が続けられなくなったときに失敗を知らせる。
   どちらの方式でも、王手でない手は詰将棋のルール違反としてその場で差し戻す。途中で方式を切り替えた場合は問題を初形へ戻す。
5. 正解手順から外れた後は、ヒントは「筋から外れている」ことだけを伝える（的外れな指示を出さない）。
6. 3手詰め・5手詰め・7手詰め・9手詰めでは、玉方が最も長く抵抗する応手を自動で指す。
7. 問題の題名・問題文は答えに直結するため盤面には表示しない（「問題N」＋汎用文のみ。2026-07-17変更）。
   題名と問題文はヒント第1段階「手筋のテーマ」として表示する。
8. ヒントは3段階: ①手筋のテーマ（旧題名＋問題文）→②駒（ハイライト＋文）→③目的地。
9. 正解手順を棋譜風のチップで表示する。
10. 問題一覧で「すべて・1手詰め・3手詰め・5手詰め・7手詰め・9手詰め」から手数を選び、同じ手数内で次の問題へ進める。未収録の手数は「準備中」と表示する。

## 問題データ

- 収録64問（1手詰め8問・3手詰め12問・5手詰め27問・7手詰め12問・9手詰め5問）は、このアプリ用に作成したオリジナル問題。インターネット上の棋譜・問題図は転載していない。複数問は、成立を確認した基本の詰み筋を左右反転・盤上の別位置へ展開した反復練習問題（飾り駒を足して配置差を作る旧方式は2026-07-17に廃止）。
- 2026-07-17、持ち駒を打つ問題への偏りを改善するため、初期持ち駒なし・攻方の全3手が盤上移動となる5手詰めを10問追加。基本筋は「竜と二枚角の盤上リレー」「成香で追い、竜で仕留める」の2種類で、成立を確認した位置違い・左右反転を反復問題として収録した。
- 2026-07-16追加の6問（突き歩成・馬・銀金の挟撃・桂金銀の隅追い・二枚桂の追い回し・二枚銀捨て）は `scripts/search-puzzles.mjs` の探索で生成し、判定エンジン自身で「初形無王手・最短手数一致・初手唯一・駒余りなし」を検証してから採用した。成り駒を使う問題（と金・馬・竜）はこの回が初収録。
- 2026-07-17、全問を不要駒（飾り駒）基準で点検し、旧作10問と「二枚桂の追い回し」の盤面を最小形に修正。飾り歩方式の反復5問と、修正で成立しなくなったミラー反復1問を廃止。代わりに新作5問（5手: 銀を取らせて呼び戻す・香を支えに銀二枚・銀を重ねて頭金、7手: 金で止めて銀で締める・隅の玉に桂の網）と「隅の玉に桂の網」のミラー反復1問を追加した（総数54問・内訳は変わらず）。
- 各問題について、次を `tests/game-core.test.mjs` で自動確認する。
  - 初期局面で玉方が王手されていない。
  - 攻め方の着手が合法である。
  - 宣言した手数が最短の詰み手数である。
  - 初手の正解となる移動元・移動先が一組だけである。
  - 段階ヒントが、玉方の最長応手を含む完全な正解手順を示す。
  - 詰め上がりで攻め方の持駒が余らない。
- 基本問題は合法手探索で完全検証し、反復問題は検証済みの基本図からの変換整合性も自動確認する。プレイ時は検証済みのヒント手順を高速経路として使い、それ以外の合法な王手は探索して判定する。

## 作問ルール（新しい問題を作る際の仕様）

すべての収録問題は、詰将棋の作法として次を満たすこと。1〜5はテストで自動保証される。

1. **初形で玉方に王手がかかっていない。**
2. **宣言した手数が最短の詰み手数である。**
3. **初手の正解が一通りしかない**（移動元・移動先の組が一組だけ）。
4. **詰め上がりで攻め方の持駒が余らない。**
5. **玉方の持駒は「盤上と攻め方の持駒を除く残り全部」**（`defenseHand: "all"`）。
6. **不要駒（飾り駒）を盤上に置かない。** 盤上の玉以外の全駒に役割があること。
   判定基準は機械的に定める: 「どの駒を1枚取り除いても、同じ唯一解・同じ手数の問題としては成立しなくなる」こと。
   `node scripts/check-minimality.mjs` で全問チェックできる（不要駒があると★で報告される）。
   反復問題に飾り駒を足して配置差を作ること（旧 extraBoard 方式）も禁止。
   ※2026-07-17、ユーザー指摘により全64問を点検し、不要駒を持つ問題をすべて最小形に修正した。

作問の手順:

1. `scripts/search-puzzles.mjs` で候補を探索する（validatePuzzle が上記1〜4と不要駒チェックを行う）。
2. `puzzles.mjs` に追加し、`puzzle-i18n.mjs` に4言語のテキストを書く。
3. `node scripts/check-minimality.mjs` と `node scripts/verify-lines.mjs` で全問を再確認する。
4. `tests/game-core.test.mjs` の問題数期待値を更新し、`npm test` → `npm run test:full` を通す。
5. `sw.js` の CACHE_NAME を上げる。

## 準拠するルール

一次情報として日本将棋連盟を参照する。

- 詰将棋のルール: https://www.shogi.or.jp/tsume_shogi/
- 将棋の対局規則: https://www.shogi.or.jp/match/taikyoku_rules/
- 通常詰将棋の駒余りの扱い: https://www.asahi-net.or.jp/~xr9s-ymgc/tume/rule.html
- 詰将棋解答選手権の駒余りを含む公式解題例: https://shogi-problem.org/2024/onlineBC/DescB.pdf

日本将棋連盟が公開する基本ルール一覧には「駒余り禁止」は明記されていない。一方、通常の創作詰将棋では、正解手順の詰め上がりで攻め方の持駒を余らせないことが完全作の要件として扱われる。本アプリもこの慣例に合わせる。

実装済み:

- 全駒の基本移動、成り、捕獲、持ち駒からの打ち
- 自玉への王手を残す手の除外
- 二歩、行き所のない駒、打ち歩詰めの除外
- 詰将棋で必要となる「玉方は盤上と攻め方の持ち駒以外の駒を合駒に使える」という前提
- 王手と詰みの判定
- 指定手数内で詰みを強制できる王手の探索
- 玉方の全応手を評価し、最長抵抗となる手を選ぶ自動応手
- 同じ最長手数の玉方応手が複数ある場合、攻め方の持駒が余らない手順を優先

現在は1手詰め・3手詰め・5手詰め・7手詰め・9手詰めに対応。長手数での探索速度と無駄合いの扱いは、問題追加時に継続して検証する。

## 主なファイル

- `index.html`: アプリ画面、設定・問題一覧・駒ガイド
- `styles.css`: iPhone向けレイアウトと3テーマ
- `app.mjs`: UI、翻訳、設定保存、問題進行
- `game-core.mjs`: 将棋の合法手、王手、詰み判定
- `puzzles.mjs`: オリジナル問題データ
- `puzzle-i18n.mjs`: 基本問題のフランス語・スペイン語テキスト
- `manifest.webmanifest`, `sw.js`: インストール・オフライン対応
- `tests/game-core.test.mjs`: 問題と判定エンジンの自動検証
- `scripts/search-puzzles.mjs`: 新問題の候補をエンジン検証付きで探索する作問支援ツール
